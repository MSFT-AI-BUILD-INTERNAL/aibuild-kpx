01왜 두 트랙을 운영하는가

"토큰을 줄이는 변환이 안전하다"를 증명하는 두 가지 서로 다른 질문에 답해야 한다:

트랙 A — 변환 자체의 건전성

kpx의 5개 SAFE_TRANSFORMS가 토큰을 정말 줄이는가, 멱등(idempotent)인가, signpost·CoT를 깎지 않는가를 외부 API 없이 빠르게 검증한다. 30 prompt × 10 transform = 300 round를 ~40 ms 안에 돌린다.

위치: bench_real/legacy/ · 키 불필요 · 항상 재현 가능

트랙 B — 실전 LLM 응답 품질

같은 변환이 실제 모델(GPT-5-mini, Claude 4.7) 응답의 품질·안전성·다국어 성능을 떨어뜨리지 않는지 측정한다. plan v2의 3-tier ladder($3 / $15 / $60)로 비용을 통제하며 6 persona별로 합격 여부를 결정한다.

위치: bench_real/ · API 키 필요 · 비용 cap 강제
트랙 A가 전제 검증(변환이 망가지지 않았는가), 트랙 B가 가치 검증(실제로 돈/품질에 이득인가)을 담당한다. A 없이 B를 돌리면 비용 낭비, B 없이 A만 보면 “토큰만 줄고 답은 나빠지는” 함정에 빠진다.

04측정·점수 산출 방법

2-tier 채점

  • Tier-1 — rule-based: expected.type으로 디스패치 (pytest · json_schema · rouge_l · chrf · refusal_label · exact · tool_choice). 모든 셀에 적용, 0–100 점.
  • Tier-2 — LLM-as-judge: 4 axes × 25점 = 100점 — correctness · completeness · faithfulness · conciseness. cross-vendor(target=GPT면 judge=Claude). Standard/Deep tier에서 20% spot-check.
  • 최종 quality = Tier-1만 있으면 그 값, 둘 다 있으면 0.5 × T1 + 0.5 × T2.

토큰 카운팅 (4종 동시)

  • tokens_in_kpx — 휴리스틱 (CJK 1글자 = 1 tok, ASCII ~4글자 = 1 tok)
  • tokens_in_tiktoken — 모델별 인코딩 (gpt-4o → o200k_base 등)
  • tokens_in_api / tokens_out_api — 응답에서 API가 보고한 실제 값

P4 calibration view가 kpx vs api 절대오차 / 상대오차를 자동 계산한다.

안전장치

CostCap

사전 가격표(PRICES)로 셀별 비용을 누계. --cap-usd N 도달 시 깨끗하게 중단. 같은 --run-id로 다시 돌리면 이미 기록된 셀을 건너뛰며 이어 실행.

Code sandbox

T01 답안은 subprocess에 -I -S로 격리 실행하고 5초 wall-timeout. 어떤 import도 사용자 시스템을 건드리지 못한다.

결정론

target/judge 모두 temperature=0, --seed 42로 spot-check 샘플링 시드 고정.

키 0개 모드

mock 어댑터 — 결정론적·과금 0 — 로 전체 파이프라인을 먼저 통과시킨 후 실제 호출로 진입. urllib만 사용해 SDK 의존성 0.

핵심 발견

1. 실제 코딩 프롬프트에서 의미 있는 절감

단순 장난감 코퍼스가 아닌 Copilot/Cursor/Aider 패턴을 모방한 프롬프트에서 V4가 EN 평균 +20.95%, 최대 +48%의 토큰 절감을 달성. bench_real/legacy의 21.28%와 독립적으로 재현됨.

2. 코드 불변수 100% 보이

수정 후 모든 SAFE variant가 fence 블록 바이트‑정확 보존, 인라인 식별자 보존, Python AST 안정성, signpost 보존, 멱등성을 100% 달성. 양적 획득과 질적 손실 0이 공존.

3. 분석 과정에서 버그 발견 · 수정 (B13)

M25가 코드 내부 들여쓰기를 제거하는 잠재 위험을 코드 보존 프로브가 잡아냄. 동일 패턴의 회귀 테스트 추가 되어 재발 차단 (현재 43/43 PASS).

4. 영한 골 구조적 확인

EN +20.95% vs KO +0.00%. KO 관용구 제거 패턴 부재가 원인. v0.3.0에서 "감사합니다", "도움이 되길 바랍니다" 등 KO 패턴 추가 예정. P6 persona가 이 신호를 매 라운드 자동 노출.

07모델별 토큰 · 비용 비교 (Opus / GPT / Gemini)

동일한 15개 실세계 코딩 프롬프트를 10개 모델(OpenAI 4 · Anthropic 3 · Google 3) × 3개 변형(V0 raw / V3 / V4 all_safe)에 교차 적용 → 쓰기 측 토큰·비용 절감을 계산한다. 총 450 cells. 동일 프롬프트가 모델 패밀리마다 다른 토탂나이저로 서로 다르게 쪼개지고, 그에 따라 kpx 축소 효과도 모델별로 조금씩 다르다는 점이 본 섹션의 핵심 관측이다.

토큰수 정확도 표기 — OpenAI는 tiktoken(o200k_base)로 정확 측정, Anthropic·Google은 공개 가이드 기반 휴리스틱(Claude 3.5 글자/토큰 · Gemini 4 글자/토큰, CJK는 1글자=1토큰). 따라서 절대값보다 추세·절감율을 보는 것이 정확하다. JSON에 tokenizer_kind 필드로 라벨 있음.
