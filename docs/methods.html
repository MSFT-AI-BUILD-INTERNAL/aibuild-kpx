2구현 상태 범례

safe transform

kpx 코드로 구현된 의미 보존 변환. SAFE_TRANSFORMS 레지스트리에 등록되어 있음.

inject

프롬프트에 텍스트를 추가하는 변환. 길이가 줄지 않을 수 있어 짧은 프롬프트에는 비권장.

comparator

변환이 아니라 직렬화 포맷 비교(JSON / YAML / MD)를 수행하는 헬퍼.

audit-only

코드 변환 없이 kpx audit가 정적 점검 → 권고만 출력.

recommendation

kpx 외부에서 처리되어야 하는 권고(어댑터 설정, 프롬프트 디자인 등).

external

외부 시스템(추론 인프라/모델 학습 측)에서만 가능한 최적화.

4안전 변환 상세 (M03 / M04 / M19 / M24 / M25 + M09)

ID함수역할주의 / 결함 이력
M03 strip_known_facts(text) "Python is a high-level interpreted language" 같은 사전학습으로 알려진 사실 줄 제거 B10 — 코드 블록 내부 보존 (_apply_outside_code)
M04 minimize_system_prompt(text) → (out, removed) "you are a helpful assistant", "do your best" 등 stock 표현 제거 B1 CoT 보존 · B3 다중절 라인 보존
M19 lossy_summary(text, max_chars=2000) 헤딩·불릿 우선 추출. flat text는 sentence boundary 자르기 B5 flat text fallback · 멱등성
M24 strip_polite(text) "Please", "Thank you", "I hope this helps" 등 politeness 제거 B2 "Please note/refer/see..." 기술 signpost 보존
M25 strip_role_tags(text) <|system|>, [INST], ### Instruction: 등 chat-template 잔여 태그 제거 B11 다중공백 정리 · B13 fenced 코드 들여쓰기 보존(R3)
M09 inject_no_filler(text) 시스템 프롬프트 끝에 "Respond with the answer only..." 1줄 추가 idempotent (substring 검사). 짧은 프롬프트엔 비권장

레지스트리 + compress 체인

SAFE_TRANSFORMS = {
    "M03": strip_known_facts,
    "M04": lambda t: minimize_system_prompt(t)[0],
    "M19": lossy_summary,
    "M24": strip_polite,
    "M25": strip_role_tags,
}

def compress(text, methods=None):
    selected = list(methods) if methods else list(SAFE_TRANSFORMS.keys())
    out = text
    for mid in selected:
        out = SAFE_TRANSFORMS[mid](out)
    out = _collapse_blank_lines(out)
    if text.endswith("\n") and not out.endswith("\n"):
        out += "\n"
    return out
핵심 invariant — 모든 안전 변환은 _apply_outside_code(text, fn) 헬퍼를 통해 fenced(```)·inline(`) 코드 블록 내부에는 절대 손대지 않는다. 이는 R3 deep analysis가 발견한 B13 결함 수정 후 전 변환에 균등 적용되었으며, 바이트 단위 동등성 회귀 테스트가 tests/test_regressions_v020.py에 추가되어 있다.
