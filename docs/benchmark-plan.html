1v1 → v2 핵심 변화

v1 — 측정 중심

  • "무엇을 측정하나" 위주, 결정 주체 모호
  • 모든 태스크 동일 가중치 (85 케이스 고정)
  • 예산 1단 ($15)만 제시 → 시작 장벽
  • 품질 임계값 단일 (-5pt) → safety 0-tolerance 미반영
  • 산출물 = 단일 report.html (페르소나별 view 없음)
  • 합의 항목 D1–D7 (장황)

v2 — 페르소나·결정 중심

  • "누가 어떤 결정을 내리는가" 출발 → 그 결정에 필요한 측정만 수행
  • 태스크는 owning persona 수로 가중 (T06 Safety는 release blocker)
  • Lite($3) / Standard($15) / Deep($60) 3-tier — Lite로 즉시 시작 가능
  • 임계값 페르소나별 분리: P3=100%, P1=−5pt, P2=총 −15%
  • raw 1개 → 페르소나별 view 6개 (CI badge / 1pager / Jupyter / md / png)
  • 합의 항목 D1–D5로 축소 (각 항목이 서로 직교)

2페르소나 P1–P6

각 페르소나는 이름 / 페인 / 결정 / 임계값 / 산출물 형태의 5-tuple로 정의된다. 이 구조를 만족하지 못하면 페르소나는 채택되지 않는다.

P1

Prompt Engineer / LLM 앱 개발자

Painkpx compress를 켜면 내 태스크의 정답률이 떨어질지 모른다. 결정각 태스크 카테고리에 어떤 transform을 default로 켤 것인가? 임계값per-task quality(Vk) ≥ quality(V0) − 5pt 산출물filterable report.html + per-task kpx compress -m ... 추천 플래그 OwnsT01, T02, T03, T04, T05
P2

Platform / FinOps 엔지니어

Pain월 LLM 청구서가 가파르게 오른다. 압축의 ROI를 입증해 게이트웨이에 default로 넣을지 결정해야 한다. 결정API 게이트웨이의 기본 변환 체인을 무엇으로 할 것인가? 임계값aggregate total_token_savings ≥ 15% AND avg quality drop ≤ 5pt 산출물1-page exec_summary.md + 월별 $ 절감 시나리오 표 Ownsaggregate(all), 특히 T03·T04 (트래픽 비중 큼)
P3

AI Safety / Compliance

Pain압축이 거절(refusal) 일관성을 깨면 컴플라이언스 사고로 이어진다. 결정kpx 압축을 production에 GREEN 줄지 RED 줄지. 단일 release blocker. 임계값refusal 보존율 = 100% (zero tolerance). 단 1건 위반 시 RED. 산출물safety_audit.json + GitHub Actions용 CI badge (pass/fail) OwnsT06 (전수)
P4

kpx maintainer (= 우리)

Painestimate_tokens() 휴리스틱이 실제 청구 토큰과 얼마나 어긋나는지 모른다. 회귀가 발생해도 자체 벤치만으로 잡히지 않는다. 결정휴리스틱 상수 보정 / default transform 순서 재조정 여부. 임계값|kpx_estimate − api_actual| / api_actual ≤ 15% (mean), idempotency 실 LLM 환경에서도 ≥ 99% 산출물calibration.md + (필요 시) tokens.py 패치 PR Ownscross-cutting (모든 셀의 token estimate delta)
P5

ML 연구자 / Karpathy 방법론 학습자

Pain30개 메서드 중 어느 것이 경험적으로 dominate하는지 정량 근거가 없다. 결정블로그 / 페이퍼에 인용할 Pareto 곡선과 framing(F1–F8)별 효과 순위. 임계값정량 임계 없음 — Pareto 곡선·CI 폭의 해석가능성이 기준 (Standard 이상에서만 충족) 산출물pareto.png + methods_per_framing.csv + Jupyter notebook OwnsV0–V5 ablation 매트릭스 전체
P6

다국어 (특히 KR/JA) Product Owner

Pain한국어/일본어는 char당 토큰 비용이 영어보다 높다. kpx의 CJK 휴리스틱이 KR 트래픽에서도 유효한지 확신 없음. 결정KR 시장 LLM 게이트웨이에 kpx를 deploy할지. 임계값T07 KR↔EN 절감률 격차 < 10pp AND CJK estimate error ≤ 20% 산출물i18n_report.md (KR/JA/ZH/EN per-lang 표) OwnsT07 (다국어)
제외된 페르소나 — Educator(임계값 없음) · Procurement($ 외 결정요인 없음) · Pen-tester(범위 ↔ 본 벤치 부적합) · DevTools/rtk integrator(v2 vNext deferred — kpx+rtk dual-side 측정은 별도 벤치)

83-Tier 캠페인

v1의 단일 $15 plan은 시작 장벽이 높다. Lite로 release 결정을 먼저 내리고, 필요할 때 Standard·Deep로 단계적 투자.

Lite

~ $3
4 시간 · 1명 · 즉시 시작 가능
  • 대상 페르소나: P1 P3
  • Tasks: T01(10) · T03(6) · T04(6) · T06(10) · T07(5) = 37 케이스
  • Variants: V0, V3, V4 (3종)
  • Models: M-A 단독 (gpt-5-mini)
  • Repeats: n=1
  • Tier 2 judge: T03만
  • 답하는 질문: "production에 default로 켜도 되는가?" (RED/GREEN)
권장

Standard

~ $15
5 일 · 1–2명
  • 대상 페르소나: P1 P2 P3 P4 P5
  • Tasks: 전체 72 케이스 (T02 5 포함)
  • Variants: V0–V5 전체 (6종)
  • Models: M-A + M-B (OpenAI + Anthropic)
  • Repeats: n=3
  • Tier 2 judge: T03 전수 + 전체 20% spot
  • 답하는 질문: 모든 페르소나의 default 의사결정 + Pareto 곡선

Deep

~ $60
2 주 · 2명 · 논문/블로그용
  • 대상 페르소나: P5 중심 + 전체
  • Tasks: 77 케이스 (T02 10 포함, 다국어 corpus 확장)
  • Variants: V0–V5 + 단일 메서드 ablation 5종 추가
  • Models: M-A + M-B + M-C(로컬 vLLM)
  • Repeats: n=5 (CI 5pt 충족)
  • Tier 2 judge: 전수 + 5% κ 일치도 (M-A·M-B 상호채점)
